HTML5 Semantic Elements

What are Semantic Elements?

Semantic elements are HTML elements that carry meaning about the structure of your web page. They clearly describe their meaning in a human- and machine-readable way. HTML5 introduced several new semantic elements to help structure your web pages better.

Common HTML5 Semantic Elements

  • <header>: Represents a container for introductory content or a set of navigational links.
  • <nav>: Defines a set of navigation links.
  • <main>: Specifies the main content of a document.
  • <article>: Defines independent, self-contained content.
  • <section>: Defines a section in a document.
  • <aside>: Defines content aside from the page content (like a sidebar).
  • <footer>: Defines a footer for a document or section.

Example: Using Semantic Elements


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Semantic HTML Example</title>
</head>
<body>
    <header>
        <h1>My Website</h1>
        <nav>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
                <li><a href="#">Contact</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <article>
            <h2>Article Title</h2>
            <p>This is the main content of the article.</p>
        </article>

        <aside>
            <h3>Related Links</h3>
            <ul>
                <li><a href="#">Link 1</a></li>
                <li><a href="#">Link 2</a></li>
            </ul>
        </aside>
    </main>

    <footer>
        <p>&copy; 2024 My Website. All rights reserved.</p>
    </footer>
</body>
</html>

Benefits of Using Semantic Elements

  • Improved accessibility for screen readers and other assistive technologies
  • Better SEO (Search Engine Optimization)
  • Easier to read and maintain code
  • More consistent structure across different websites
